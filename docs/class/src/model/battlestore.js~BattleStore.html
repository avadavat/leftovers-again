    public data(): Object source

    Output function for getting an object representation of the current battle.

    This is usually called right after 'handleRequest', because requests usually mean we're trying to get the battle's state and make some decisions based on it.

    This function has NO EFFECT on the store's internal state; this is important for consistency!

    Return:

    Object

    An object representing the battle's current state. See the documentation in AI for a detailed description.

    public handleRequest(json: String) source

    Handles an incoming request. The one parameter to this is a string of JSON, known as the request.

    what does the request look like? WELL. These properties are all integrated into the AI object, so you probably want to look at that instead. But in case you're wondering what the actual data from the server looks like, keep reading.

    MoveData objects here are limited and contain only 'move' (the move name, ex. 'Fake Out', 'id' ex. 'fakeout', 'pp', 'maxpp', 'target', and 'disabled'.

    PokemonData objects are limited and contain only 'ident', 'details', 'condition', 'hp', 'maxhp', 'active', 'stats', 'moves', 'baseAbility', 'item', 'pokeball', and 'canMegaEvo'.

    With most of this information, we may know the things already, ex. we know if a Pokemon took damage or not. However there are lots of ways we would get out of sync if we didn't accept and process all of this information. So we do that.

    You'll notice the request does not contain information about the opponent! It's left as an exercise to US to keep track of the opponent's state.

    Params:

    NameTypeAttributeDescription
    json String

    The string of JSON which makes up the request.
    json.active Array<Object>

    An array containing the moves that your active Pokemon can perform. The size of the array is the number of active Pokemon on your side, ex. in Singles matches, the array length is 1.
    json.active[].moves Array<MoveData>

    (the 4 moves of your active pokemon)
    json.forceSwitch Array<Boolean>

    booleans for each position that needs to switch out. ex. [true] means it's a singles match and your mon needs to switch out. [false, true] means it's a doubles match and your second mon needs to switch out.
    json.noCancel Boolean

    Moves cannot be cancelled in the interval between sending the move and the server receiving your opponent's move. This is unused.
    json.rqid String

    The request ID. ex. '1' for the first turn, '2' for the second, etc. These don't match up perfectly with turns bc you may have to swap out pokemon if one dies, etc.
    json.side Object
    json.side.id String

    either 'p1' or 'p2'
    json.side.name String

    your name
    json.side.pokemon Array<PokemonData>

    6 of them. they're the pokemon on yr side.
    json.wait Boolean

    True if this is not a request - just updated information. The opponent needs to do something; ex. if their mon feinted last turn, they need to choose a mon to send in. This is unused.
