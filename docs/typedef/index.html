    public MoveData: * source

    Properties:

    NameTypeAttributeDescription
    accuracy Number | Boolean

    The move's accuracy, as a percent. 100 or true means they will always connect, unless affected by something else.
    basePower Number

    The base power, ex. 80.
    category String

    'Physical', 'Special', or 'Status'
    flags Object

    From the flags Showdown server:
    flags.authentic Boolean

    Ignores a target's substitute.
    flags.bite Boolean

    Power is multiplied by 1.5 when used by a Pokemon with the Ability Strong Jaw.
    flags.bullet Boolean

    Has no effect on Pokemon with the Ability Bulletproof.
    canZMove Boolean

    True if this move has an available Z-move.
    flags.charge Boolean

    The user is unable to make a move between turns.
    flags.contact Boolean

    Makes contact.
    flags.defrost Boolean

    Thaws the user if executed successfully while the user is frozen.
    flags.distance Boolean

    Can target a Pokemon positioned anywhere in a Triple Battle.
    flags.gravity Boolean

    Prevented from being executed or selected during Gravity's effect.
    flags.heal Boolean

    Prevented from being executed or selected during Heal Block's effect.
    flags.mirror Boolean

    Can be copied by Mirror Move.
    flags.nonsky Boolean

    Prevented from being executed or selected in a Sky Battle.
    flags.powder Boolean

    Has no effect on Grass-type Pokemon, Pokemon with the Ability Overcoat, and Pokemon holding Safety Goggles.
    flags.protect Boolean

    Blocked by Detect, Protect, Spiky Shield, and if not a Status move, King's Shield.
    flags.pulse Boolean

    Power is multiplied by 1.5 when used by a Pokemon with the Ability Mega Launcher.
    flags.punch Boolean

    Power is multiplied by 1.2 when used by a Pokemon with the Ability Iron Fist.
    flags.recharge Boolean

    If this move is successful, the user must recharge on the following turn and cannot make a move.
    flags.reflectable Boolean

    Bounced back to the original user by Magic Coat or the Ability Magic Bounce.
    flags.snatch Boolean

    Can be stolen from the original user and instead used by another Pokemon using Snatch.
    flags.sound Boolean

    Has no effect on Pokemon with the Ability Soundproof.
    id String

    The move ID, ex. 'acrobatics'
    name String

    The move name, ex. 'Acrobatics'
    priority Number

    Does this move have priority? Most have the value 0. Moves with priority 1 will go before moves with priority 0 in normal cases.
    self Object

    Does this have an effect on myself?

    status String

    target String

    Ex. 'normal', 'self', 'allySide', 'any', 'randomNormal', 'all', 'allAdjacent', allAdjacentFoes', 'foeSide'
    type String

    The type of move, ex. 'Ghost'. Every move has one and only one type.
    volatileStatus String

    A volatile status, if there is one. (ex. 'protect' or 'taunt').
    zMove MoveData

    The Z-move move data.

    public PokemonData: * source

    Properties:

    NameTypeAttributeDescription
    ability String

    The mon's ability, if known.
    abilities Object

    A key-value list of abilities the Pokemon might have. The keys are numbers ('0', '1', etc.) plus the key 'H' for hidden abilities.
    active Boolean

    True if this mon is active, undefined otherwise.
    baseAbility String

    The mon's ability, if known. This will only be set for Pokemon you control.
    baseStats Object

    An object with these properties:
    baseStats.atk: Number

    The attack value before boosts, EVs, IVs, etc.
    baseStats.def: Number

    The defense value before boosts, EVs, IVs, etc.
    baseStats.spa: Number

    The special attack value before boosts, EVs, IVs, etc.
    baseStats.spd: Number

    The special defense value before boosts, EVs, IVs, etc.
    baseStats.spe: Number

    The speend value before boosts, EVs, IVs, etc.
    boosts Object

    An object with properties set for boosts and unboosts. For example, if this mon has cast Swords Dance, you will have boosts = {atk: 2}. Boost values range from -6 to 6.
    canMegaEvo Boolean

    True if this Pokemon can mega-evolve. This is set on active Pokemon only, not Pokemon in your reserve.
    canZMove Array

    Exists if this Pokemon can use a Z-move. This is set on active Pokemon only, not Pokemon in your reserve. Z-move stuff is set on the Move itself as well.
    condition String

    a condition in Showdown format, ex. '100/100 par poi', '0/100 fnt' etc.
    dead boolean

    True if the mon is dead, undefined otherwise
    events Array<Object>

    Not currently being used. (things that happened to this mon? things this mon did?)
    id *

    a mon's id, in Showdown format. @TODO does this exist? probs not
    gender String

    One of 'M', 'F', or undefined/empty(?)
    hp Number

    The mon's current HP.
    hppct Number

    The mon's current HP as a percentage of max HP, in case that's easier to use.
    level Number

    the level of the mon.
    maxhp Number

    The mon's max HP. Note that if the 'HP Percentage Mod' rule is set, this will be 100 for all of your opponent's mons.
    moves Array<MoveData>

    an array of Moves.
    nature String

    The mon's nature, ex. 'Jolly'. Usually, you only know this about your own mons.
    owner String

    The mon's owner, ex. 'p1' or 'p2'
    position String

    The mon's position, in Showdown format. 'p1a' means they are in player 1's first active slot; 'p2c' means they are in player 2's third active slot (in a Triples battle)
    prevMoves Array<String>

    An array containing the move ids for moves this Pokemon has used. prevMoves[0] is the last used move, and prevMoves[prevMoves.length-1] is the first move that Pokemon used when they were out on the field. This array is reset when a Pokemon switches out.
    types Array<String>

    An array of the mon's types, ex. ['Fire', 'Flying']
    seenMoves Array<String>

    An array of moves that we've seen this Pokemon use. This carries throughout the match. The string is the move id.
    species String

    the species of Pokemon, ex. "Pikachu". This is the same as {@link PokemonData.id|PokemonData.id}, but more human-readable.
    stats Object

    An object similar to baseStats, but includes calculations based on EVs, IVs, and level. It does NOT include calculations based on boosts and unboosts.
    statuses Array<String>

    an array of status conditions, ex. ['par', 'poi'].
    weightkg Number

    The mon's weight, in kg.
